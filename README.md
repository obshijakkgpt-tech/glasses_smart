# CyanBridge

Голосовой ассистент-приложение для Android: говорит с вашим сервером **Hermes**
и (опционально) управляет смарт-очками **HeyCyan**. Тёмный минималистичный
интерфейс на Jetpack Compose.

Работает в двух режимах:
- **Без очков** (по умолчанию) — телефон как микрофон и динамик. Ничего, кроме
  телефона и доступа к вашему Hermes, не нужно.
- **С очками HeyCyan** — управление по Bluetooth и выгрузка медиа по WiFi
  (требует дозаполнения протокола, см. ниже).

## Как собрать APK — два пути

### Путь 1 (рекомендуется): GitHub Actions — сборка в облаке

Ничего ставить на компьютер не нужно. APK соберётся на серверах GitHub
бесплатно. Подробная пошаговая инструкция — в файле **ИНСТРУКЦИЯ_GITHUB.md**.
Кратко:

1. Создай бесплатный аккаунт и **новый репозиторий** на github.com.
2. Загрузи туда содержимое этой папки (через кнопку «uploading an existing file»).
3. Открой вкладку **Actions** — сборка запустится сама.
4. Когда появится зелёная галочка, скачай **CyanBridge-debug-apk** внизу
   страницы запуска (раздел Artifacts).
5. Перекинь `app-debug.apk` на телефон и установи.

Рецепт сборки уже лежит в проекте: `.github/workflows/build.yml`.

### Путь 2: Android Studio (локально)

Если предпочитаешь IDE — см. **ИНСТРУКЦИЯ_ANDROID_STUDIO.md**.
Коротко: установить Android Studio → Open → выбрать эту папку → дождаться
синхронизации → меню Build → Build APK(s) → Locate.

## 4. Первая настройка (чтобы заговорило с вашим Hermes)

Откройте приложение → вкладка **Настройки**:

| Поле | Что вписать |
|------|-------------|
| **Адрес сервера** | URL вашего Hermes, например `http://192.168.1.50:8642` |
| **API-ключ** | значение `API_SERVER_KEY` из вашего Hermes |
| **Модель** | как настроено на сервере (поле косметическое для Hermes) |
| **Язык распознавания** | `ru-RU`, `en-US` или `auto` |
| **Язык озвучки** | `ru-RU`, `en-US` или `auto` |
| **Режим** | «Без очков (телефон)» для старта |

Нажмите **Проверить соединение** — должно быть «Соединение успешно».

> На стороне Hermes должен быть включён HTTP API-сервер:
> `API_SERVER_ENABLED=true` и задан `API_SERVER_KEY`. Это OpenAI-совместимый
> сервер (`/v1/chat/completions`, `/health`), по умолчанию порт **8642**.
> Телефон и сервер должны быть доступны по сети (одна сеть / VPN / проброс).

Теперь на вкладке **Главная** нажмите большую кнопку микрофона, разрешите
доступ к микрофону и говорите. Состояния «Слушаю… / Распознаю… / Думаю… /
Озвучиваю…» показывают, на каком шаге процесс.

## 5. Структура проекта

```
cyanbridge-redesign/
├── build.sh / build.bat        # сборка APK без IDE
├── app/                        # Android-приложение (Kotlin, Compose)
│   └── src/main/java/com/cyanbridge/app/
│       ├── ui/                 # экраны: home, notes, media, settings + тема
│       ├── voice/              # STT/TTS телефона + VoicePipeline + состояния
│       ├── hermes/             # HermesClient (HTTP к вашему серверу)
│       ├── data/               # Room (история, очередь задач) + настройки
│       ├── domain/             # GlassesController (контракт) + Fake-реализация
│       ├── ble/                # реальный BLE-клиент очков + таблица протокола
│       └── net/                # WiFi-выгрузка медиа (Ogg/Opus, HTTP, MediaStore)
├── backend/                    # ОПЦИОНАЛЬНО: референс-стек (не нужен, если
│                               #  Hermes у вас уже развёрнут)
└── PROJECT_MEMORY.md           # журнал разработки и принятых решений
```

## 6. Режим очков HeyCyan (опционально, для позже)

Управление очками по Bluetooth готово на уровне «скелета»: сканирование,
подключение, очередь команд, выгрузка медиа по WiFi с упаковкой Opus в Ogg и
сохранением в галерею. Не хватает только **точных байтов BLE-команд** —
они изолированы в одном файле:
```
app/src/main/java/com/cyanbridge/app/ble/HeyCyanProtocol.kt
```
Их нужно перенести из ветки `manufacturer-original` оригинального репозитория
HeyCyan SDK (места помечены `TODO(protocol)`). После этого режим «Очки
HeyCyan» в Настройках заработает с реальным устройством без изменений в UI.

## 7. Частые проблемы

- **«Java не найдена»** — установите JDK 17 (см. п.1) и перезапустите терминал.
- **«Соединение успешно» не появляется** — проверьте, что Hermes доступен с
  телефона (откройте `http://АДРЕС:8642/health` в браузере телефона), и что
  включён `API_SERVER_ENABLED`.
- **Распознавание речи не работает** — на телефоне должен быть установлен
  движок распознавания (Google). Проверьте язык в Настройках.
- **Сборка падает на SDK** — удалите файл `local.properties` и запустите
  скрипт заново; либо задайте `ANDROID_HOME` вручную.
