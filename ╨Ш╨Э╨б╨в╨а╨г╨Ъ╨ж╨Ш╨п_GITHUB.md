# Как собрать APK через GitHub (для новичка, по шагам)

Тебе НЕ нужно ничего устанавливать на компьютер. APK соберётся на серверах
GitHub бесплатно. Нужен только браузер.

Весь процесс — минут 10 в первый раз. Дальше каждая новая сборка идёт сама.

---

## Шаг 1. Создай аккаунт GitHub (если ещё нет)

1. Зайди на https://github.com
2. Нажми **Sign up**, заведи аккаунт (бесплатно).

---

## Шаг 2. Создай новый репозиторий

«Репозиторий» — это просто папка с проектом на сайте GitHub.

1. В правом верхнем углу нажми **+** → **New repository**.
2. **Repository name**: впиши, например, `cyanbridge`.
3. Поставь точку **Private** (приватный — виден только тебе) или **Public** —
   неважно, оба работают.
4. НЕ ставь галочки «Add a README» и т.п. — оставь репозиторий пустым.
5. Нажми зелёную кнопку **Create repository**.

---

## Шаг 3. Загрузи файлы проекта

Сейчас у тебя есть архив `CyanBridge-redesign.zip`. **Сначала распакуй его**
на компьютере (правый клик по zip → «Извлечь всё»). Получится папка с файлами.

1. На странице пустого репозитория найди ссылку
   **uploading an existing file** (в тексте «Get started by … uploading an
   existing file»). Нажми её.
2. Открой на компьютере распакованную папку.
3. **Выдели ВСЁ содержимое** папки (Ctrl+A) и перетащи мышью в окно браузера
   в зону «Drag files here».
   - Важно: перетаскивай содержимое (папки `app`, `gradle`, `.github`,
     файлы `build.gradle.kts` и т.д.), а не саму внешнюю папку.
   - Если папка `.github` не перетащилась (некоторые системы прячут папки
     с точкой) — ничего, ниже в Шаге 6 есть запасной вариант.
4. Внизу нажми зелёную кнопку **Commit changes**.
5. Подожди, пока всё загрузится (может занять минуту — файлов много).

---

## Шаг 4. Запусти сборку

1. Вверху страницы репозитория открой вкладку **Actions**.
2. Если увидишь предложение включить Actions — нажми
   **I understand my workflows, go ahead and enable them**.
3. Сборка с названием **Build APK** должна запуститься автоматически.
   Если нет — слева выбери **Build APK** и нажми **Run workflow** → **Run workflow**.

---

## Шаг 5. Дождись результата и скачай APK

1. Зайди в идущую сборку (строка с жёлтым кружком = идёт, зелёная галочка =
   готово, красный крест = ошибка).
2. Дождись зелёной галочки (первый раз ~3–6 минут).
3. На странице завершённой сборки прокрути вниз до раздела **Artifacts**.
4. Скачай **CyanBridge-debug-apk** — внутри будет `app-debug.apk`.
5. Распакуй и перекинь `app-debug.apk` на телефон (кабель/Telegram/облако).
6. На телефоне открой файл и установи (разреши «установку из неизвестных
   источников», если попросит).

Готово — приложение на телефоне.

---

## Шаг 6. Если папка `.github` не загрузилась

Файл с рецептом сборки лежит по пути `.github/workflows/build.yml`. Если он
не загрузился (папки с точкой иногда прячутся), создай его прямо на сайте:

1. На главной странице репозитория нажми **Add file** → **Create new file**.
2. В поле имени впиши ровно:
   ```
   .github/workflows/build.yml
   ```
   (слэши сами создадут папки)
3. В большое поле вставь содержимое файла `build.yml` из проекта
   (открой его в распакованной папке любым текстовым редактором — например
   «Блокнотом» — и скопируй всё).
4. Нажми **Commit changes**.
5. Вернись к Шагу 4.

---

## Если сборка упала (красный крест)

Не страшно — это чинится. Открой упавшую сборку, нажми на красный шаг,
скопируй текст ошибки и пришли его мне в чат. Я подскажу, что поправить.

Самые частые причины: не загрузилась папка `.github` (см. Шаг 6) или
загрузилась не вся папка `app`. В этом случае проще удалить репозиторий
(Settings → внизу Delete) и повторить Шаг 2–3 аккуратнее.
